# Merge Audit: with-property → real-estate-project

**Base (main):** `real-estate-project` (the IR site — "WITH / Pham Tri: Sovereign Capital, Zone 4 Can Tho Investor Relations")
**Incoming:** `clone-snapshot-dev-main` (the "With Property" brokerage micro-site, route `/with-property`)
**Goal:** merge the With Property page into the IR site as route `/with-property`, and add a nav button on the IR site that navigates to it.

## 1. Stack compatibility

Both projects are TanStack Start + React 19 apps generated by Lovable, with identical `package.json` dependency versions (`@tanstack/react-router` 1.168.x, `@tanstack/react-start` 1.167.x, React 19.2, Tailwind 4.2.1, sonner, lucide-react, etc.), identical `vite.config.ts`, `tsconfig.json`, and `components.json`. Only two trivial package.json diffs: the clone has `@lovable.dev/vite-tanstack-config` pinned to `2.5.3` vs `^2.5.2` in main, and main has an extra `framer-motion` dependency the clone doesn't use. No structural incompatibility — this is a low-risk merge.

## 2. Routing

Main currently has a single route, `/` (`src/routes/index.tsx`), rendering the IR homepage. The clone has `/` (its own placeholder homepage — not needed) and `/with-property` (`src/routes/with-property.tsx`), which renders the full With Property page (`Nav`, `Hero`, `WhyChoose`, `Segments`, `Services`, `About`, `LeadForm`, `Footer`) wrapped in its own `LangProvider`. Plan: copy `with-property.tsx` into main's `src/routes/`, drop the clone's `index.tsx`, and let TanStack Router regenerate `routeTree.gen.ts` (auto-generated, don't hand-edit).

## 3. File collisions (same path, incompatible contents)

These four files exist at the identical path in both projects but are different, non-interchangeable implementations. This is the core merge risk:

- **`src/components/site/LangContext.tsx`** — Main's version is a simple `{lang, setLang}` context with a separate `t(lang)` translation lookup. The clone's version is a richer context that also exposes `t(key)` directly, persists language choice to `localStorage`, and types `Lang` from `translations.ts` rather than defining it locally.
- **`src/components/site/translations.ts`** — Main's dictionary is keyed by section (`nav.links`, `hero.title1`, `macro.items`, etc.) for the IR site's content. The clone's dictionary is a flat key-value map (`nav_why`, `hero_title`, etc.) for With Property's content. These are two unrelated translation schemas, not a superset/subset.
- **`src/components/site/Reveal.tsx`** — Main implements scroll reveal with `framer-motion`. The clone implements the same effect with a hand-rolled `IntersectionObserver` + CSS class (`wp-reveal`/`is-visible`) and has no `framer-motion` dependency.
- **`src/components/site/SectionHeader.tsx`** — Different prop shapes (`subtitle`/`dark`/`center` in main vs `description`/`align` in the clone) and different markup/styling.

**Resolution:** since `With Property` only consumes these from its own component tree (`src/components/with-property/*`), the safest fix is to namespace the incoming versions instead of overwriting main's: e.g. copy them as `LangContext.tsx`, `translations.ts`, `Reveal.tsx`, `SectionHeader.tsx` into a new `src/components/with-property-shared/` (or similar) folder, and update the with-property components' imports to point there. Do not overwrite the `site/` versions — that would break the existing IR homepage.

## 4. Files with no collision (safe to copy as-is)

- `src/components/with-property/*` (About, Footer, Hero, LeadForm, Nav, Segments, Services, WhyChoose) — new folder, no name clash.
- `src/routes/with-property.tsx` — new route file.
- `src/assets/wp-hero.jpg` — new asset, no name clash.
- All `src/components/ui/*` shadcn primitives are byte-identical between the two projects (same generator) — no action needed, main's copies are sufficient.

## 5. Styling conflict

`src/styles.css` defines two incompatible design-token systems under the same CSS variable names: main uses hex colors (`--primary: #14a76c`, custom `--gold`/`--emerald-brand`/`--panel` tokens, Inter/Poppins fonts) while the clone uses the shadcn-default oklch palette plus its own `--wp-*` tokens (`--wp-navy`, `--wp-gold`, `--wp-border`, etc.) and a `wp-theme`/`wp-serif`/`wp-reveal` class scheme. Because the With Property page wraps itself in a `.wp-theme` root div and scopes its colors to `--wp-*` variables, the cleanest fix is to append the clone's `--wp-*` variable block, `.wp-theme` scoped overrides, `wp-serif`, `wp-gold-rule`, and `wp-reveal`/`is-visible` classes onto the end of main's `styles.css`, rather than replacing main's `:root`/`.dark` blocks. Needs a visual check after merge to confirm `.wp-theme` properly scopes and doesn't leak into the IR site's global styles.

## 6. Root shell (`src/routes/__root.tsx`)

Differs only in `<head>` metadata (title/description/og tags — main's is already correct, "WITH" branding) and font links (main loads Inter+Poppins from Google Fonts; the clone relies on its own CSS for fonts). No merge conflict — keep main's `__root.tsx` unchanged. The With Property route brings its own `<LangProvider>` wrapper scoped to just that page, so it doesn't depend on root's global `LangProvider`.

## 7. The actual ask: button from IR site → With Property page

IR site nav lives in `src/components/site/Nav.tsx`, driven by `NAV_LINKS` in `src/components/site/data.ts`. Today the nav's right-hand action is a disabled "Secure Login" button that opens a dialog. Add a `Link to="/with-property"` (from `@tanstack/react-router`) as a new nav button — e.g. next to or in place of the secure-login button — labeled something like "With Property" / "Brokerage Services". Mirror the same `Link to="/with-property"` in the mobile drawer menu in the same file.

## 8. Pre-existing repo issue (unrelated to this merge, flagging for cleanup)

`real-estate-project/real-estate-project/real-estate-project/...` is a real-estate-project repo nested inside itself two levels deep, each level a full git checkout (`.git`, `node_modules`, etc.). This looks like an accidental `git clone` run inside the working copy rather than anything from this merge. Recommend deleting the nested `real-estate-project/real-estate-project` directory before or after this merge — it's dead weight and risks being mistaken for the real source tree.

## 9. Suggested merge order

1. Delete the stray nested `real-estate-project/real-estate-project` folder (item 8).
2. Copy `src/components/with-property/` and `src/assets/wp-hero.jpg` into main as-is.
3. Create `src/components/with-property-shared/` (or chosen name) with the clone's `LangContext.tsx`, `translations.ts`, `Reveal.tsx`, `SectionHeader.tsx`; repoint `with-property/*` imports to it.
4. Copy `src/routes/with-property.tsx` into main's `src/routes/`; update its imports for step 3's new path.
5. Append the clone's `--wp-*` tokens / `.wp-theme` / `wp-serif` / `wp-reveal` CSS to the end of main's `src/styles.css`.
6. Add the nav button + mobile-menu link in `src/components/site/Nav.tsx` pointing to `/with-property`.
7. Run the dev server, regenerate `routeTree.gen.ts`, and visually verify both `/` and `/with-property` render correctly with no style bleed.
